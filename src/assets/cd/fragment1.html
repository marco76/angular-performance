By design Angular activate for each Component and his children the automatic change detection. This feature increase the number of events generated and checked by Angular. To reduce the number of events generated the automatic change detection should be deactivated: 

    @Component({
      changeDetection: ChangeDetectionStrategy.OnPush
    })

The markforChange is called:

  • After an event in the DOM: click, submit, ...
  • After the data is fetched via Http: click, submit, ...
  • Activated by timers: setTimeout, setInterval

Dev vs. Prod mode

If started in Dev mode the Angular application does more checks than in the Prod mode. Be sure to deploy the Prod version to your users and don't use only the Dev app to evaluate the performance.
